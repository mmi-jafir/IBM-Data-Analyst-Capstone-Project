This repository contains the **IBM Data Analyst Capstone Project**, the final course in the **IBM Data Analyst Professional Certificate** program offered on Coursera.

In this project, I applied various data analytics techniques to analyze real-world datasets and identify **technology trends in the global developer community**.

The project simulates the role of an **Associate Data Analyst** working for a global IT consulting firm. The objective is to collect, prepare, analyze, and visualize data to identify **current and emerging technology skill trends**.

---

# Project Overview

Technology companies must continuously track emerging tools and skills to remain competitive.  
In this project, developer survey data is analyzed to identify trends in:

- Programming languages  
- Databases  
- Platforms and cloud technologies  
- Web frameworks  
- Developer demographics  

The analysis is based on a subset of the **Stack Overflow Developer Survey dataset**, which contains responses from developers around the world.

---

# Dataset

The dataset originates from the **Stack Overflow Developer Survey**, one of the largest surveys of software developers globally.

Dataset source:  
https://stackoverflow.blog/2019/04/09/the-2019-stack-overflow-developer-survey-results-are-in/

This capstone project uses a **randomized subset of the full dataset**.

**Note:** Because the dataset is a subset, insights should be interpreted as **indicative trends rather than exact real-world statistics**.

---

# Project Workflow

The project follows a complete **data analytics workflow** consisting of six modules.

---

## Module 1: Data Collection

Data was collected from multiple sources including APIs, web pages, and survey datasets.

Tasks included:

- Collecting data using APIs  
- Collecting data using web scraping  
- Exploring and understanding the collected data  

---

## Module 2: Data Wrangling

The collected data was cleaned and prepared for analysis.

Tasks included:

- Identifying missing values  
- Handling missing data  
- Detecting duplicate records  
- Removing duplicates  
- Normalizing and formatting data  

---

## Module 3: Exploratory Data Analysis (EDA)

Exploratory analysis was performed to identify patterns and relationships within the dataset.

Analysis included:

- Distribution analysis  
- Outlier detection  
- Correlation analysis  

---

## Module 4: Data Visualization

Data visualizations were created to communicate insights and trends effectively.

Examples include:

- Distribution charts  
- Scatter plots for relationship analysis  
- Composition charts (pie charts)  
- Comparison charts (bar charts and line charts)

---

## Module 5: Dashboard Creation

Interactive dashboards were created to present insights dynamically.

Tools used:

- **Google Looker Studio**
- **IBM Cognos Analytics**

The dashboards highlight:

- Current technology trends  
- Future technology trends  
- Developer demographics  

---

## Module 6: Presentation of Findings

The final stage involved summarizing insights and presenting the results through a structured presentation.

The presentation highlights:

- Key technology trends  
- Emerging skills and technologies  
- Developer demographic patterns

# Key Insights

Key insights from the analysis include:

- **JavaScript, SQL, and Python remain among the most widely used programming languages.**
- **Cloud platforms such as AWS, Azure, and Google Cloud dominate modern development environments.**
- **PostgreSQL and MySQL remain widely used relational databases.**
- Developers show growing interest in modern technologies such as **React, Node.js, Go, and Rust**.
- The developer workforce is largely **young, globally distributed, and highly educated**.
